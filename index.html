firewall.js is a node.js based full-featured firewall that leverages the blazing fast asyncronous V8 engine at the heart of the node.js platform to perform any and all firewall function required.
Rules are persisted in standard JSON format making it extremely easy to work with and interoperable with thousands of already existing applications.
By moving the firewall functionality closer to the application filtering decisions can be made in conjunction between the application and the firewall, rules can be determined in real-time by application state.

No more lengthy processes to alter firewall rules, they are all determined by the application they are protecting, making them more relevant and up-to-date than ever.
The firewall can be instantiated both globally and application-private, making it possible to double the protection by both having a server-wide firewall and a private application firewall at the same time.
Rules and logs can be altered and accessed using an interactive web console, secured by https and aes 256 bit cryptography.

FAQ

How secure is firewall.js?
Even more secure than traditional firewalls. By using modern platforms like javascript to implement the firewall, instead of old-time languages like C or C++, vulnerabilities like buffer overflows are completely eliminated.
How is firewall.js distributed?
As a standard NPM package. All you need to do is installing it, no more extra costly servers and complex configurations.
